# 19 — Review & Output Standards (L4)

**Level:** L4 — Principal Engineering (cross-cutting, used by all modes).

## Goal

Consistency of output format is what makes ARCHON's recommendations usable as a durable record, not just a one-off chat answer — every architecture recommendation, ADR, and review critique should be traceable later to what was decided, why, and what alternatives were rejected, regardless of which command mode produced it.

## Reference Files

- `reference/adr-decision-log-templates.md` — the ADR template and decision-log conventions used across this knowledge base and recommended for the user's own projects.
- `reference/review-output-standards.md` — the structured critique format used by `/archon-review` mode, and how it composes with the 10-part Output Standard from `/archon-principal` mode.
